About Cpp

C++各个基本变量的大小（64位）

单位均位Byte 1Byte=8bit

int 4,float 4,long 4,double 8,char 1,short 2,bool 1

任意类型的指针8（指针存储的是一个地址，64位机的地址64bit=8Byte）

引用和指针的区别

引用是一个变量的别名不是实体故不能独立存在，在使用引用时必须初始化，而指针不需要初始化

example

int a=0; int &b=a; (√)

int &a;(×)

自增运算时不同

example

int a=0,*b=&a;

b++;实质上是内存指向的区域往后便宜了一个int的内存大小

int a=0,&b=a;

b++;此时b=1;

指针可以为空而引用不可以

new和malloc

同样的问题还有delete和free

首先malloc和free均为函数，而new和delete均为运算符

new 应该和delete搭配，new []应该和delete []搭配

malloc和free搭配

malloc不能重载而new可以重载operator new（new也不能重载噢）

malloc调用时需要自行计算分配到内存大小，而new可以省略这一步

malloc返回的内存区域实际上是(void*),而new会依据自动转换成对应的类型

关于声明和定义

extern int a;//声明了一个a变量

int a;//定义了一个a变量

重载与重写

重载发生在同一个类内部，函数名相同但参数列表不同（可能返回参数也不相同）

重写发生在基类和派生类之间，且要求基类对应函数应该声明为virtual，依靠虚函数可以做到运行时多态

C++程序的编译过程

1.预处理：做一些宏定义的展开替换操作

2.编译：进行语义语法分析，生产汇编语句

3.汇编：生成机器码

4.链接：链接成可执行的exe文件

注：模板是在编译阶段确认类型的

运行时多态如何实现

1.要求基类对应的方法必须声明为virtual方法

2.此时基类和父类的实际内存大小会比按照变量的计算多8Byte（64位机）->多出一个虚指针

3.在编译时会生成虚函数表（存放在常量区）

4.调用对应方法时依靠虚指针到虚函数表中查找真正应该执行的方法

指针数组与数组指针等傻傻分不清的问题

int (*p)[10];//一个指针，指向一个有10个元素的整型数组

int *p[10];//一个数组，存放了十个整型的指针

int (*p)(int,int);//一个指针，指向一个有两个整数作为输入参数，一个整数作为输出参数的函数（函数地址是函数名哦）

int(*p(int int))[10];//一个函数指针，指向输入位两个整数，输出位有十个整数的数组的函数

typedef int arr[10]; arr* fun(int ,int);//同上

using int[10] arr;arr* fun(int,int);//同上

Cast(Src)

1.static_cast<>()

​ 只要没有底层const，就可以用此转换

​ 可以关闭一些精度损失警告，转换（void *）

2.const_cast<>()

​ 专用于转换有底层const的变量

3.reinterpret_cast<>()

​ 很危险的操作，用于提供更加底层的转化

4.dynamic_cast<>()

​ 用于进行运行时多态的一些转换

​ 常见的输入有：

​ dynamic_cast<*>(e);

​ dynamic_cast<&>(e);

​ dynamic_cast<&&>(e); //&&用于右值

左值和右值

int a;//左值

int a,b;int c=a*b;//a，b，c均为左值，而a*b为右值（易得右值仅为一个零时的数据，不占用内存资源）

构造函数与析构函数

1.构造函数不能设置为virtual，此时虚函数表尚未填入对应的方法，故不能实现多态

2.析构函数默认为是可以重写的，且一般建议设置为virtual（防止部分编译器没默认允许析构函数重写）

3.除构造函数以外的一切非static函数，均可以设置为virtual
