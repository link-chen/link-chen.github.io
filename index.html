OS

操作系统是什么

是一套软件，运行再应用软件和硬件之间的软件，用于管理计算机的资源。

操作系统管理的资源有哪些

处理器，内存，文件，输入输出

操作系统发展阶段：无操作系统，批处理系统，分时操作系统，实时操作系统，互联网阶段的操作系统

处理器管理

进程与线程：进程是程序运行的一次动态运行，用进程控制块PCB唯一标识一个进程。进程由PCB，数据段，代码段共同组成。进程是资源分配的单位，一个进程中可以有多个线程，线程是处理器调用的基本单位。

进程的状态：创建，就绪，运行，阻塞，终止

（后期补一个状态转移的图）

进程间的通信问题

管道，共享内存，消息队列（1），电子邮件（2），不同设备上的进程可以通过socket通信

（1）（2）是消息传递的直接和间接方式

处理机调度

（1）作业调度：只会有一次，再开启一个程序时，从硬盘加载到程序中

（2）中级调度：从内存中调度到硬盘的虚拟内存，或从虚拟内存中调入内存

（3）进程调度：大量平凡的调度，使用时间片轮转等算法进行调度

进程调度算法

（1）时间片轮转

（2）短作业优先

（3）高相应比

（4）先来先服务

（5）优先级调度

线程的同步和互斥方法

1.信号量

2.锁（互斥锁和读写锁）

3.条件变量

死锁的必要条件

不可剥夺，互斥访问，条件保持，循环等待

银行家算法：最大可分配矩阵，需求矩阵，资源矩阵

内存管理

操作系统运行在内存的低地址段，应用程序运行在高地址段

现代操作系统中，程序可以动态重定位装入内存中（主要是应用了程序的局部性原理cache缓存）

内存的分配方式：动态分区分配，非连续性内存分配

动态分区分配

1.首次适应算法：地址升序，找第一个符合需求的内存区域

2.最佳适应算法：内存大小升序，找第一个符合需求的内存区域

3.最差适应算法：内存大小降序，找第一个符合需求的内存区域

4.邻次适应算法：地址升序，从上一次分配的区域开始，找第一个符合需求的内存区域

非连续性分配方式

1.分页

​ 每一个页都是固定大小的，地址由页号+页内偏移量组成

​ 通过每个地址内的页号在页表内找到该页的对应物理块，然后块数*每块大小+页内偏移量=真实在内存中的地址。如果没有在页表内找到对应的页号，则发生缺页中断

​ 二级页表

2.分段

​ 段号+段内偏移量，段号长度和段内偏移量都不定长

​ 依据段号查段表，找到段基址，基址+段内偏移量=内存实际地址

3.段页式内存管理

页面置换算法

1.OPT(最佳置换算法)，不切实际但确实是最好的方法（因为不能预测未来的情况）

2.FIFO，会出现贝拉德异常（即可放置页面的内存增加，缺页异常反而增加的情况）

3.LRU，将最久没有使用的页面置换出去

4.CLOCK置换算法

引用和指针的区别

引用是一个变量的别名不是实体故不能独立存在，在使用引用时必须初始化，而指针不需要初始化

example

int a=0; int &b=a; (√)

int &a;(×)

自增运算时不同

example

int a=0,*b=&a;

b++;实质上是内存指向的区域往后便宜了一个int的内存大小

int a=0,&b=a;

b++;此时b=1;

指针可以为空而引用不可以

野指针相关

一个指针p，在定义后没有初始化，就会变成一个野指针

野指针指向的区域未知，但确实会指向一段内存区域

对于一个野指针p

p==nullptr//得到的值是0，即此时野指针p不为空

为了程序的安全，要防止出现野指针

new和malloc

同样的问题还有delete和free

首先malloc和free均为函数，而new和delete均为运算符

new 应该和delete搭配，new []应该和delete []搭配

malloc和free搭配

malloc不能重载而new可以重载operator new（new也不能重载噢）

malloc调用时需要自行计算分配到内存大小，而new可以省略这一步

malloc返回的内存区域实际上是(void*),而new会依据自动转换成对应的类型

凡是new就必须delete，否则容易内存泄漏（当然可以考虑智能指针）

重载与重写

重载发生在同一个类内部，函数名相同但参数列表不同（可能返回参数也不相同）

重写发生在基类和派生类之间，且要求基类对应函数应该声明为virtual，依靠虚函数可以做到运行时多态

C++程序的编译过程

1.预处理：做一些宏定义的展开替换操作

2.编译：进行语义语法分析，生产汇编语句

3.汇编：生成机器码

4.链接：链接成可执行的exe文件

注：模板是在编译阶段确认类型的

运行时多态如何实现

1.要求基类对应的方法必须声明为virtual方法

2.此时基类和父类的实际内存大小会比按照变量的计算多8Byte（64位机）->多出一个虚指针

3.在编译时会生成虚函数表（存放在常量区）

4.调用对应方法时依靠虚指针到虚函数表中查找真正应该执行的方法

指针数组与数组指针等傻傻分不清的问题

int (*p)[10];//一个指针，指向一个有10个元素的整型数组

int *p[10];//一个数组，存放了十个整型的指针

int (*p)(int,int);//一个指针，指向一个有两个整数作为输入参数，一个整数作为输出参数的函数（函数地址是函数名哦）

int(*p(int int))[10];//一个函数指针，指向输入位两个整数，输出位有十个整数的数组的函数

typedef int arr[10]; arr* fun(int ,int);//同上

using int[10] arr;arr* fun(int,int);//同上

Cast<Target*>(Src)

1.static_cast<>()

​ 只要没有底层const，就可以用此转换

​ 可以关闭一些精度损失警告，转换（void *）

2.const_cast<>()

​ 专用于转换有底层const的变量

3.reinterpret_cast<>()

​ 很危险的操作，用于提供更加底层的转化

​ interpret是诠释的意思，那顾名思义reinterpret就是重新诠释这个区域

​ 依据此方法进行的类型转换，会在指针的底层解引用成Target类型的

​ 例如大端存储的int a=0x61620000 本质上是整数，但如果通过reinterpret转换，打印的a会是“ab”(详情参见ASCII码)

4.dynamic_cast<>()

​ 用于进行运行时多态的一些转换

​ 常见的输入有：

​ dynamic_cast<*>(e);

​ dynamic_cast<&>(e);

​ dynamic_cast<&&>(e); //&&用于右值

构造函数与析构函数

1.构造函数不能设置为virtual，此时虚函数表尚未填入对应的方法，故不能实现多态

2.析构函数默认为是可以重写的，且一般建议设置为virtual（防止部分编译器没默认允许析构函数重写）

3.除构造函数以外的一切非static函数，均可以设置为virtual

拷贝构造函数和移动构造函数

首先明确哪些情况下会使用拷贝构造函数？

1.有等号发生时（需要重载赋值运算符支持这一行为）

2.函数传入的是一个类，但既不是指针也不是引用时

1.使用std::move(t);//t是被移动的类

2.移动构造函数是来骗，来偷袭，直接抢走t的部分内存区域的指针，并将t的指针指向区域为null

关于define和const定义的值

define使一个宏而不是变量，将会在编译的预处理阶段进行宏展开，将值进行替换，不会占用内存

const是定义了一个常量，实际上在内存中占有一段区域

函数调用的过程

+++++++++ 入栈 ++++++++++++
将实参从右向左压入栈
压入返回地址
压入主调函数的基地址
跳到被调用函数的地址，执行函数代码，局部变量按声明顺序依次压入栈
将返回值放入寄存器eax(累加器)中
+++++++++ 出栈 ++++++++++++
局部变量全部出栈
返回地址出栈，找到原执行地址
形参出栈
赋值操作将寄存器中的返回值赋给左值（如果有的话）

virtual 如何实现运行时多态

如果一个类中定义了virtual方法，那么对该类调用sizeof，会发现比不带virtual的方法多8字节（64位机器上）。实际上多出来的8字节是一个指针（虚指针），指向虚函数表（存放在常量区）。运行时依据虚指针在虚函数表中查找真正的方法，达到运行时多态，解决了二义性的问题
